                                     1                                          Let’s talk about imVision RESTFul API.                    
 
                                              An API is an application programming interface.        This is an interface that enables controlled interaction between different software programs.        In other words, the imVision API allows to develop integration programs between other software packages and imVision system.                                2                 
 
                                              Two authentication methods are supported to access data via API.       They are Token and basic authentication.       By default,      Token based authentication is the only method enabled. The Token expiration time is configurable in this page, it is set to 60 minutes by default.          You can additionally enable basic authentication.                                3                 
 
                                              Here we see a quick      demo of SM authentication settings, and in this case we choose to have both Token and Basic authentication enabled.                                4                 
 
                                              The first thing to use the API with token      authentication is to get a token.        In order to get the token, we use basic authentication and send a valid SM username password (in this example admin/Admin1) in a token request call.        System Manager replies with a unique token that is valid for 60 minutes in this case.       Now we can start sending API calls using this token instead of having to send our user name and password in each call.          Also, if the user is deleted from SM, any active tokens are automatically expired, so a deleted user can no longer use the assigned token, even if the token’s expiration time has not yet come.           Once we have the token we use API get buildings function to get the list of all the buildings in our database and their information.                                   5                 
 
                                              Notes:          The restful API comes active      (does not need to be enabled).             Can API function be disabled? NO, the API cannot be turned OFF from SM. Y          ou can stop the imVisionAPIPool in IIS’s Application Pools to disable it.                Can token authentication be disabled?            No, it cannot be disabled.                           Can the listening port be changed?                            You can change the port in IIS’s Web Site Bindings. But this will change the port used by the whole web site, including System Manager Web interface.                        Can API be made secure?                            You can use https instead of http, but you need to add the https in IIS’s Web Site Bindings.                       Is there a privilege associated?                  No privilege in SM is today related to API feature. Any user account can request token or do basic auth                 If the account is deleted in SM, the token is revoked immediately                                8
